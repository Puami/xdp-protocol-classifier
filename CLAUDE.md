# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A set of [containerlab](https://containerlab.dev/) topologies demonstrating dual-stack (IPv4 + IPv6) networking with custom Docker images. The primary learning goals are containerlab topology authoring, Docker-based network nodes, clean separation of image vs. node-specific configuration, and IP routing.

## Key Commands

**Build the Docker image:**
```bash
bash scripts/build-image.sh basic-lab
bash scripts/build-image.sh bind-entrypoint-lab
bash scripts/build-image.sh routing-lab
```

**Deploy a topology (from lab directory):**
```bash
cd containerlab/basic-lab && ./deploy.sh
cd containerlab/bind-entrypoint-lab && ./deploy.sh
cd containerlab/routing-lab && ./deploy.sh
```

**Inspect / verify:**
```bash
cd containerlab/basic-lab
containerlab inspect -t basic-lab.clab.yml
docker exec clab-basic-lab-node1 ip addr show
```

**Test connectivity:**
```bash
docker exec clab-basic-lab-node1 ping -c 3 10.0.0.2       # IPv4
docker exec clab-basic-lab-node1 ping -6 -c 3 fc00::2     # IPv6
```

**Shell access:**
```bash
docker exec -it clab-basic-lab-node1 bash
```

**Destroy topology (from lab directory):**
```bash
cd containerlab/basic-lab && ./destroy.sh
```

## Architecture

### Lab structure

Each lab lives in its own subdirectory under `containerlab/`. Shared deploy/destroy logic lives in `containerlab/lib/` and is sourced by thin 5-line wrappers in each lab directory. Each lab has its own Docker image tag so labs are fully independent.

| Lab | Image | Entrypoint | Nodes |
|-----|-------|------------|-------|
| `basic-lab` | `clab-softnet-basic:latest` | baked into image via `COPY` | 2 (node1, node2) p2p |
| `bind-entrypoint-lab` | `clab-softnet-bind-ep:latest` | bind-mounted from host at runtime | 2 (node1, node2) p2p |
| `routing-lab` | `clab-softnet-routing:latest` | bind-mounted from host at runtime | 3 (hs1, rt1, hs2) linear |

### Lifecycle separation

The Dockerfile uses `CMD ["sleep", "infinity"]` as PID 1 — the container stays alive indefinitely. Network configuration (IP assignment, etc.) is done **after** containerlab creates the virtual links, via the `exec` directive in the topology file. This avoids polling for interface readiness: `eth1` is guaranteed to exist when the exec command runs.

### Node-agnostic entrypoint

`bin/entrypoint.sh` contains no hostname logic or hardcoded addresses. All node-specific values (`NODE_IP`, `NODE_IP6`, `PEER_IP`, etc.) come from a config file bind-mounted at `/etc/nodes/<hostname>.cfg`. The topology file mounts the right config for each node read-only (`:ro`).

### IPv6 DAD disabled

Addresses are added with the `nodad` flag to skip Duplicate Address Detection, eliminating ~1 s startup delays (safe in a lab where addresses are pre-assigned and unique).

### Config file format (`configs/node*.cfg`)

Plain bash variables sourced by the entrypoint:
```bash
NODE_IP=10.0.0.1
NODE_PREFIX=24
NODE_IP6=fc00::1
NODE_PREFIX6=64
PEER_IP=10.0.0.2
PEER_IP6=fc00::2
```

### Runtime state

`clab-<lab-name>/` directories are auto-generated by containerlab on deploy (Ansible inventory, TLS artifacts, topology JSON). They are gitignored and should not be edited manually.

## Adding a New Lab

1. Create `containerlab/<lab-name>/` with a `Dockerfile`, topology file, `bin/entrypoint.sh`, and `configs/`
2. Write `deploy.sh` and `destroy.sh` wrappers that set `LAB_NAME`, `IMAGE`, `TOPOLOGY`, `NODES`, `LAB_DIR` and source `../lib/deploy.sh` / `../lib/destroy.sh`
3. Each lab uses its own image tag (`clab-softnet-<lab-name>:latest`) to stay fully independent
